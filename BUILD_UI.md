# GroceryFlow IMS — FTXUI TUI Upgrade

## Quick Start (Windows + MSVC)

### Prerequisites
1. **Visual Studio 2022** (or later) with "Desktop development with C++" workload
2. **vcpkg** for dependency management
3. **CMake** (3.20+)

### Setup Steps

#### 1. Install & Integrate vcpkg
```bash
# Clone vcpkg (if not already done)
git clone https://github.com/microsoft/vcpkg.git

# Bootstrap
cd vcpkg
.\bootstrap-vcpkg.bat

# Integrate with Visual Studio (one-time)
.\vcpkg integrate install

# Install FTXUI for x64 Windows
.\vcpkg install ftxui:x64-windows
```

#### 2. Configure & Build with CMake
```bash
cd c:\Users\faris\OneDrive\Desktop\WORKSHOP 1\Project\smart grocery

# Configure (generates Visual Studio project)
cmake -S . -B build `
  -DCMAKE_TOOLCHAIN_FILE=<path-to-vcpkg>\scripts\buildsystems\vcpkg.cmake `
  -A x64

# Build in Release mode
cmake --build build --config Release
```

#### 3. Run the UI Demo
```bash
.\build\Release\main_ui.exe
```

**Hotkeys in the demo:**
- **q** — Quit
- **↑/↓** — Navigate sidebar menu
- **→/←** — Tab between sidebar and main content

## Troubleshooting

### Error: `ftxui not found`
- Ensure vcpkg is installed and integrated: `.\vcpkg integrate install`
- Verify the triplet is correct: `.\vcpkg list` should show `ftxui`

### Error: `CMAKE_TOOLCHAIN_FILE not found`
- Check the path to `vcpkg.cmake` — it should be at: `<vcpkg-root>/scripts/buildsystems/vcpkg.cmake`
- If using a different location, adjust the `-DCMAKE_TOOLCHAIN_FILE` path

### Build fails with missing `libftxui-*`
- Rebuild FTXUI: `.\vcpkg install ftxui:x64-windows --force`

### Terminal renders incorrectly
- FTXUI works best in **Windows Terminal** or **ConEmu** (not legacy cmd.exe)
- Terminal size must be ≥80×24 characters
